# 边缘设备的实时水下视觉定位系统及方法

## 软件设计说明文档

本设计说明文档用于描述“边缘设备的实时水下视觉定位系统及方法”的软件总体设计、模块组成、关键算法、运行流程及实施细节，面向软件著作权申请材料编制。文档内容结合本项目在 `asr_sdm_video_inertial_odometry` 包内的实际实现，涵盖依赖环境、模块接口、运行流程与调试方式，并配合流程图文件 `svo1.0_pipeline.svg` 与 `msckf_fronted_pipeline.svg` 说明系统处理链路。

### 1.2 软件开发目的

针对现有水下视觉定位方案在边缘设备部署中存在的算法计算开销大、纯视觉鲁棒性不足、系统扩展性差、固定融合权重不适应时变环境等问题，开发一款可直接部署于低算力边缘设备的实时水下视觉定位软件，实现水下机器人在复杂水下环境中的稳定、精准定位，降低工程部署成本，提升定位系统的鲁棒性与扩展性。

### 1.3 软件核心功能

1. 模式切换：支持单目/双目相机模式切换，适配不同水下场景需求
2. 传感器采集与预处理：采集图像流与 IMU 数据，完成时间同步、图像增强/去畸变、IMU 去噪与预积分
3. 半直接视觉前端：基于像素残差与稀疏特征联合约束估计相对位姿，降低计算开销
4. 多源融合估计：以 EKF/滤波框架融合视觉与 IMU 先验，输出稳定的位姿估计
5. 多传感器扩展：预留统一观测接口，支持深度计、DVL、声呐等外部传感器增量接入
6. 位姿输出：输出六自由度位姿、速度与置信度，供导航控制使用
7. 运行与调试：提供 ROS2 节点、话题及 RViz2 可视化能力

## 二、技术领域与背景技术

### 2.1 技术领域

本软件属于水下机器人导航与定位技术领域，具体涉及一种面向低算力嵌入式边缘设备的实时水下视觉定位系统及方法，尤其涉及以半直接视觉前端与扩展卡尔曼滤波为核心、支持单目/双目切换与多传感器扩展接入的模块化轻量化定位软件。

### 2.2 背景技术

水下机器人在巡检、测绘、管道检测、应急搜救等场景中对连续定位能力要求极高。现有水下视觉定位方案在边缘设备部署过程中普遍存在计算开销大、鲁棒性不足、扩展困难、固定融合权重难以适应时变环境等问题。因此需要一种低计算开销、高鲁棒性、易扩展、可直接部署于低算力边缘设备的实时水下视觉定位软件。

## 三、软件总体设计

### 3.1 设计原则

1. 轻量化：降低视觉前端计算复杂度，适配低算力边缘平台
2. 模块化：核心模块解耦，便于维护与扩展
3. 鲁棒性：视觉退化时仍能依靠 IMU 先验保持连续定位
4. 可扩展性：统一观测接口，支持多传感器增量接入
5. 实时性：保证图像与 IMU 高频数据稳定处理

### 3.2 总体架构

软件以 `svo` 与 `svo_ros` 为核心视觉前端与 ROS2 接入层，配合 `vikit_common` 工具库完成相机模型、数学与图像处理封装。总体流程为：

- 传感器数据采集
- 时间同步与预处理
- 半直接视觉前端
- IMU 预积分
- 滤波融合
- 位姿解算输出

系统软件流程图见 `svo1.0_pipeline.svg`，前端处理流程图见 `msckf_fronted_pipeline.svg`。

### 3.3 模块组成及功能详解

#### 3.3.1 视觉采集与 ROS2 接口模块（svo_ros）

通过 ROS2 订阅 `/camera/image_raw`、IMU 话题等数据源，完成相机图像流与惯性数据输入。提供 `test_rig3.launch.py`、`test_euroc.launch.py`、`vio_euroc.launch.py` 等启动文件完成参数加载与节点启动。

#### 3.3.2 IMU 采集与校正模块

读取 IMU 角速度与加速度数据，进行时间同步、零偏估计与尺度因子校正，并完成预积分，为滤波预测提供运动先验。

#### 3.3.3 半直接视觉前端模块（svo）

基于像素残差与稀疏特征联合约束估计相机相对位姿。模块包含特征提取、稀疏对齐、重投影与位姿优化（对应 `feature_detection`、`sparse_img_align`、`pose_optimizer` 等实现），保证低算力下的实时性。

#### 3.3.4 时间同步与预处理模块

完成图像与 IMU 数据时间对齐，对图像进行去畸变与增强，对 IMU 数据进行去噪、异常剔除与预积分，输出标准化数据供前端与滤波模块使用。

#### 3.3.5 滤波融合后端模块

以 IMU 运动模型为预测方程，以视觉位姿观测为更新方程，输出位姿与速度估计。状态向量定义为：

X = [px, py, pz, qw, qx, qy, qz, vx, vy, vz, ba, bg]^T

#### 3.3.6 动态权重更新模块

基于视觉残差趋势、IMU 残差趋势与视差一致性，在线调节视觉与 IMU 融合比例，改善水下环境下定位稳定性。

#### 3.3.7 扩展接口模块

预留统一观测接口，可增量接入深度计、DVL、声呐、磁罗盘等传感器，以附加观测形式参与融合。

#### 3.3.8 位姿解算与输出模块

输出 `/svo/pose`、`/tf` 等 ROS2 话题，提供六自由度位姿、速度与置信度信息，供导航与控制使用。

## 四、关键算法与数据模型

### 4.1 半直接视觉前端算法

以像素残差为核心约束，结合稀疏特征点匹配进行位姿优化，避免密集特征提取与匹配带来的高计算开销。关键流程为：

- 图像预处理
- 稀疏特征提取
- 像素残差计算
- 位姿优化
- 视觉位姿初估

### 4.2 滤波融合算法

采用扩展卡尔曼滤波思想：

1. 预测阶段：IMU 运动模型驱动状态预测
2. 更新阶段：视觉位姿观测更新系统状态
3. 状态向量：X = [px, py, pz, qw, qx, qy, qz, vx, vy, vz, ba, bg]^T

### 4.3 动态权重更新

视觉残差 ev、IMU 残差 ei、视差一致性 ηd ∈ [0, 1] 共同决定融合权重：

- λv = exp(-β|Δev|)(1 - ηd) + 1
- λi = exp(γ|Δei|)

当视觉质量下降时增大视觉协方差，提高 IMU 权重；当 IMU 误差增长时提高视觉权重。

### 4.4 数据模型

1. 图像数据：输入分辨率 640×480，帧率 20fps，可根据配置调整
2. IMU 数据：输入角速度/加速度，频率 100~400Hz
3. 融合输出：位姿（px, py, pz, qw, qx, qy, qz）、速度（vx, vy, vz）及置信度
4. 扩展传感器：可选深度计/DVL 数据按统一观测接口接入

## 五、软件运行流程

### 5.1 运行流程概述

系统启动后进行模式选择、标定加载与时间同步，随后执行视觉前端与 IMU 预积分，进入滤波融合与动态权重更新，输出位姿信息并循环执行。流程图见 `svo1.0_pipeline.svg` 与 `msckf_fronted_pipeline.svg`。

### 5.2 详细运行步骤

S1：初始化，加载相机/IMU 标定、设置模式（单目/双目）、完成时间同步与 EKF 初值

S2：采集图像与 IMU 数据并预处理

S3：半直接视觉前端估计视觉位姿

S4：IMU 预积分形成高频运动先验

S5：滤波预测与更新完成融合

S6：动态权重更新并输出位姿/速度/置信度

S7：循环执行 S2 至 S6

### 5.3 运行操作说明（ROS2）

#### 5.3.1 依赖安装
```sh
sudo apt-get -y install ros-$ROS_DISTRO-sophus libgoogle-glog-dev
```
```sh
colcon build --symlink-install --parallel-workers 8
```

#### 5.3.2 软件构建
```sh
colcon build --packages-up-to svo_ros
```
#### 5.3.3 运行步骤（推荐三终端）

1. 环境配置（每个终端）：
```sh
   source install/setup.bash
```


2. 终端1：启动 SVO 节点：
```sh
   ros2 launch svo_ros test_rig3.launch.py fast_type:=9
   ros2 launch svo_ros test_rig3.launch.py fast_type:=10
   ros2 launch svo_ros test_rig3.launch.py fast_type:=11
   ros2 launch svo_ros test_rig3.launch.py fast_type:=12
   ros2 launch svo_ros test_euroc.launch.py
   ros2 launch svo_ros vio_euroc.launch.py
```

3. 终端2：播放 rosbag：
```sh
   ros2 bag play datasheet/airground_rig_s3_ros2
   ros2 bag play ~/svo/asr_sdm_ws/datasheet/MH_01_easy_ros2 --clock --rate 1.0
```

4. 终端3：启动 RViz2：
```sh
   rviz2 -d src/asr_sdm_universe/perception/asr_sdm_video_inertial_odometry/asr_sdm_svo/svo_ros/rviz_config.rviz
```

#### 5.3.4 调试与监控命令
```sh
ros2 node list
ros2 topic list | sort
ros2 node info /svo
ros2 topic hz /svo/image
ros2 topic hz /svo/pose
```
#### 5.3.5 进程停止命令
```sh
pkill -f "ros2 bag play"
pkill -f "ros2 launch svo_ros test_rig3.launch.py"
pkill -f "rviz2 -d .*rviz_config.rviz"
```

## 六、具体实施方式

### 6.1 实施例一：单目 + IMU 实时定位

1. 硬件配置：低功耗边缘处理器、单目防水相机、IMU
2. 软件配置：ROS2（Rolling/Jazzy）、svo/svo_ros、vikit_common、Sophus、glog
3. 运行参数：相机分辨率 640×480，帧率 20fps；IMU 100Hz；关键点数 100~200
4. 运行效果：低算力下稳定运行，视觉退化时保持轨迹连续

### 6.2 实施例二：双目 + IMU + 深度计融合

1. 硬件配置：低功耗边缘处理器、双目相机、IMU、深度计（CAN 接口）
2. 软件配置：启用双目模式，扩展深度计观测接口
3. 运行参数：双目 20fps；深度计 10Hz；关键点数 200~300
4. 运行效果：提升垂向稳定性，抑制累计漂移

## 八、摘要

本软件公开了一种边缘设备的实时水下视觉定位系统及方法。软件以半直接视觉前端降低计算开销，结合 IMU 预积分与滤波融合实现稳定定位，并通过动态权重更新策略提升水下时变环境的鲁棒性。系统支持单目/双目切换与多传感器扩展接入，可部署于低算力边缘平台，满足水下机器人巡检、测绘、应急搜救等场景的实时定位需求。
