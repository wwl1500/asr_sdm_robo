# ASR_SDM_VIDEO_ENHANCEMENT

This node subscribes to a camera topic and then uses image enhancement algorithms to improve the image quality.

## Topics

- Input Topic
  - `/asr_sdm_video_enhancement/debug/image/compressed` (`sensor_msgs::msg::CompressedImage`)

- Output Topic
  - `/asr_sdm_video_enhancement/output/image` (`sensor_msgs::msg::Image`, use `compressed` transport in `rqt_image_view` when you want JPEG)

Internally the C++ enhancement node uses `~/debug/image/compressed` and `~/output/image`.

## Parameters

- airlight
  - The default value is `255`; a smaller value results in a brighter processed image.
- scale
  - This parameter allows you to scale the original image to ensure real-time processing.
- transmission_floor
  - The default value is `0.25`; a larger value keeps the restoration more conservative.
- transmission_gain
  - The default value is `0.50`; a smaller value reduces dehazing strength.
- gamma_blend
  - The default value is `0.30`; a smaller value pulls contrast correction closer to neutral.
- white_balance_blend
  - The default value is `0.35`; white balance is blended with the dehazed image instead of fully replacing it.
- clahe_blend
  - The default value is `0.30`; CLAHE is applied as a light mix instead of a full replacement.
- original_blend
  - The default value is `0.20`; the output mixes some original detail back in to protect structure.
- clahe_clip_limit
  - The default value is `1.4`; lower than before to avoid over-enhancement.
- overlay_fps_text
  - The default value is `true`; the current FPS is drawn on the output image in real time.

Realtime defaults stay conservative. The offline tools also support a separate
`offline_gt_like` quality profile for brighter, clearer exports with offline-only
pre-denoising.

## MCAP Playback

Use the C++ MCAP player to replay a rosbag2 directory backed by MCAP storage.

```bash
ros2 run asr_sdm_video_enhancement asr_sdm_mcap_player --ros-args \
  -p bag_path:=/tmp/water_mcap_test
```

Default parameters:

- `bag_path=/tmp/water_mcap_test`
- `loop=false`
- `playback_hz=0.0` (`0` means follow bag timestamps; set `20.0` for fixed 20 Hz replay)
- The node replays the original bag topic name unchanged
- The node only supports bags that contain exactly one `sensor_msgs/msg/CompressedImage` topic

The player follows the timestamps stored in the bag. The first message is published immediately, and later messages are replayed using the relative timestamp gaps stored in the MCAP file.

For the current bags generated by the converter script, the replayed topic is typically:

```bash
/asr_sdm_video_enhancement/debug/image/compressed
```

The node republishes `sensor_msgs/msg/CompressedImage` directly. It does not decode compressed frames into raw `sensor_msgs/msg/Image`.

To replay at a fixed rate instead of the original bag timing:

```bash
ros2 run asr_sdm_video_enhancement asr_sdm_mcap_player --ros-args \
  -p bag_path:=/tmp/water_mcap_test \
  -p playback_hz:=20.0
```

## MP4 To MCAP

Use the converter script to generate a rosbag2 directory backed by MCAP storage.

```bash
ros2 run asr_sdm_video_enhancement convert_mp4_to_mcap -- \
  --output /tmp/water_mcap
```

This writes a standard rosbag2 directory that contains an MCAP file and `metadata.yaml`, so it can be inspected and replayed directly:

```bash
ros2 bag info /tmp/water_mcap
ros2 bag play /tmp/water_mcap
```

## Evaluation

Use the offline evaluator to compare the original blur videos with your enhanced output against the
paired reference videos under `/home/cortin/Downloads/test`.

Reference data convention:

- `blur/cv_<id>.mp4`
- `gt/cv_<id>_<method>.mp4`

The `gt` videos are treated as manually selected reference videos, not strict physical ground truth.
The evaluator reports:

- Reference metrics: `PSNR`, `SSIM`
- No-reference underwater metrics: `UCIQE`, `UIQM`

Default evaluation command:

```bash
ros2 run asr_sdm_video_enhancement asr_sdm_video_enhancement_eval
```

This uses:

- `--blur-dir=/home/cortin/Downloads/test/blur`
- `--reference-dir=/home/cortin/Downloads/test/gt`
- `--output-dir=/tmp/asr_sdm_video_eval`
- `--quality-profile=default`
- `--frame-step=1`
- `--metric-scale=1.0`

The evaluator still accepts `--airlight` and `--scale` as optional overrides. An
explicit `--airlight` override disables adaptive airlight estimation for that run.

For faster exploratory runs on high-resolution videos, sample every 5th or 10th frame and
downscale metric computation, for example:

```bash
ros2 run asr_sdm_video_enhancement asr_sdm_video_enhancement_eval \
  --blur-dir /home/cortin/Downloads/test/blur \
  --reference-dir /home/cortin/Downloads/test/gt \
  --output-dir /tmp/asr_sdm_video_eval_fast \
  --quality-profile offline_gt_like \
  --frame-step 10 \
  --metric-scale 0.5
```

Generated files:

- `/tmp/asr_sdm_video_eval/pair_metrics.csv`
- `/tmp/asr_sdm_video_eval/video_summary.csv`
- `/tmp/asr_sdm_video_eval/method_summary.csv`
- `/tmp/asr_sdm_video_eval/overall_summary.json`

Interpretation:

- `delta_psnr` / `delta_ssim` compare `enhanced -> reference` against `input -> reference`
- `delta_uciqe` / `delta_uiqm` compare `enhanced` against `input`
- Use `PSNR/SSIM` as similarity-to-reference indicators, and use `UCIQE/UIQM` to support the
  claim that underwater visual quality improved

## Batch Export

Use the offline exporter to enhance every paired blur video and write the processed videos to an
output directory as MP4 files.

Default export command:

```bash
ros2 run asr_sdm_video_enhancement asr_sdm_video_enhancement_export \
  --blur-dir /home/cortin/Downloads/test/blur \
  --reference-dir /home/cortin/Downloads/test/gt \
  --output-dir /home/cortin/Downloads/test/output
```

To generate a brighter, offline-only GT-like export profile:

```bash
ros2 run asr_sdm_video_enhancement asr_sdm_video_enhancement_export \
  --blur-dir /home/cortin/Downloads/test/blur \
  --reference-dir /home/cortin/Downloads/test/gt \
  --output-dir /home/cortin/Downloads/test/output_compare \
  --quality-profile offline_gt_like
```

Behavior:

- Only `blur/cv_<id>.mp4` files that have a matching `gt/cv_<id>_<method>.mp4` are exported
- Unmatched or unsupported blur file names are skipped and recorded in `export_summary.csv`
- Output files keep the original blur file names and are written as MP4 using the `mp4v` codec
- Output videos keep the original blur FPS and frame size; if `scale < 1.0`, processed frames are
  resized back to the original resolution before writing
- `offline_gt_like` enables stronger dehazing, lighter original-frame blending, and offline-only
  `fastNlMeansDenoisingColored` pre-denoising to suppress suspended-particle noise

Generated file:

- `/home/cortin/Downloads/test/output/export_summary.csv`

## Demo Overlay

Use the demo overlay script to generate presentation-friendly `cv_1045` MP4 files with a
top-right metrics panel.

Default command:

```bash
ros2 run asr_sdm_video_enhancement annotate_cv1045_demo
```

Generated files:

- `/home/cortin/Downloads/test/demo_overlay/cv_1045_blur_annotated.mp4`
- `/home/cortin/Downloads/test/demo_overlay/cv_1045_enhanced_annotated.mp4`

The overlay shows:

- Video title (`Origin` or `Enhanced`)
- Reference label (`cv_1045_LANet`)
- Current frame index
- Video FPS
- Current frame size
- Fixed PSNR / SSIM / UCIQE / UIQM values taken from the existing single-video summary JSON
